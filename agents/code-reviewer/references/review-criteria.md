---

## 기본 리뷰 체크리스트

### 보안 (CRITICAL 후보)
- [ ] SQL Injection 취약점
- [ ] XSS (Cross-Site Scripting)
- [ ] 인증/인가 우회 가능성
- [ ] 민감 정보 노출 (API 키, 비밀번호, 토큰)
- [ ] 안전하지 않은 역직렬화
- [ ] 경로 탐색 (Path Traversal)

### 안정성 (CRITICAL/MAJOR 후보)
- [ ] 무한 루프 가능성
- [ ] 메모리 누수
- [ ] 널 포인터 역참조
- [ ] 경쟁 조건 (Race Condition)
- [ ] 리소스 미해제 (파일, 연결 등)

### 로직 (MAJOR 후보)
- [ ] 비즈니스 로직 오류
- [ ] 엣지 케이스 미처리
- [ ] 잘못된 조건문
- [ ] 오프바이원 에러 (Off-by-one)

### 성능 (MAJOR/MEDIUM 후보)
- [ ] N+1 쿼리
- [ ] 불필요한 반복 연산
- [ ] 대용량 데이터 메모리 로드
- [ ] 비효율적인 알고리즘

### 코드 품질 (MEDIUM/LOW 후보)
- [ ] 중복 코드
- [ ] 과도한 복잡성 (Cyclomatic Complexity)
- [ ] 긴 함수/메서드
- [ ] 매직 넘버/문자열
- [ ] 부적절한 에러 처리

### 유지보수성 (LOW 후보)
- [ ] 네이밍 컨벤션
- [ ] 주석 부족/과잉
- [ ] 일관성 없는 코드 스타일
- [ ] 사용하지 않는 코드

## 리뷰 시 참고사항

1. **컨텍스트 이해**: 변경의 목적과 배경을 파악
2. **영향 범위 확인**: 변경이 다른 부분에 미치는 영향
3. **테스트 커버리지**: 변경에 대한 테스트 존재 여부
4. **문서화**: API 변경 시 문서 업데이트 여부
