# TC 설계 패턴

> 효과적인 테스트 케이스 설계를 위한 패턴 가이드

---

## 1. 경계값 분석 (Boundary Value Analysis)

입력 필드의 경계값을 집중 테스트합니다.

### 예시: 나이 입력 (1-120)

| TC | 입력값 | 예상 결과 |
|----|-------|----------|
| BVA-01 | 0 | 유효성 에러 |
| BVA-02 | 1 | 정상 처리 |
| BVA-03 | 60 | 정상 처리 |
| BVA-04 | 120 | 정상 처리 |
| BVA-05 | 121 | 유효성 에러 |

### 예시: 비밀번호 (8-20자)

| TC | 입력값 | 예상 결과 |
|----|-------|----------|
| BVA-01 | 7자 | 유효성 에러 |
| BVA-02 | 8자 | 정상 처리 |
| BVA-03 | 20자 | 정상 처리 |
| BVA-04 | 21자 | 유효성 에러 |

---

## 2. 동등 분할 (Equivalence Partitioning)

입력을 유효/무효 클래스로 분류하여 대표값 테스트합니다.

### 예시: 이메일 입력

**유효 클래스:**
- 표준 형식: `user@domain.com`
- 서브도메인: `user@mail.domain.com`
- 플러스 태그: `user+tag@domain.com`

**무효 클래스:**
- @ 누락: `userdomain.com`
- 도메인 누락: `user@`
- 특수문자: `user@@domain.com`

---

## 3. 상태 전이 테스트 (State Transition)

상태 변화 흐름을 테스트합니다.

### 예시: 주문 상태

```
[주문접수] → [결제대기] → [결제완료] → [배송준비] → [배송중] → [배송완료]
                ↓                                           ↓
           [주문취소]                                   [반품요청]
```

**테스트 케이스:**
- 정상 흐름: 주문접수 → ... → 배송완료
- 취소 흐름: 결제대기 → 주문취소
- 반품 흐름: 배송완료 → 반품요청
- 잘못된 전이: 배송중 → 결제대기 (불가)

---

## 4. 페어와이즈 테스트 (Pairwise Testing)

다중 조건 조합을 효율적으로 테스트합니다.

### 예시: 검색 필터 조합

| 카테고리 | 정렬 | 가격대 | 배송 |
|---------|------|-------|------|
| 전자기기 | 최신순 | 전체 | 무료배송 |
| 의류 | 인기순 | ~5만원 | 일반배송 |
| 식품 | 낮은가격 | 5~10만원 | 퀵배송 |

모든 조합(81개) 대신 핵심 페어(~15개)만 테스트합니다.

---

## 5. 유즈케이스 테스트 (Use Case)

실제 사용자 시나리오 기반 테스트입니다.

### 예시: 쇼핑몰 구매 시나리오

**Happy Path:**
1. 상품 검색
2. 상품 상세 확인
3. 장바구니 담기
4. 결제 진행
5. 결제 완료
6. 주문 확인

**Alternative Path:**
- 3a. 바로 구매 (장바구니 스킵)
- 4a. 쿠폰 적용
- 4b. 포인트 사용

**Exception Path:**
- 4x. 결제 실패 → 재시도
- 5x. 재고 소진 → 알림

---

## 6. 에러 추측 (Error Guessing)

경험 기반으로 에러 가능성이 높은 케이스를 테스트합니다.

### 일반적인 에러 케이스

**입력 관련:**
- 빈 값 제출
- 공백만 입력
- SQL Injection 시도 (`' OR 1=1--`)
- XSS 시도 (`<script>alert(1)</script>`)
- 매우 긴 문자열 (10000자)

**네트워크 관련:**
- 느린 네트워크 (3G 시뮬레이션)
- 요청 중 네트워크 끊김
- 중복 클릭 (빠른 연타)

**상태 관련:**
- 세션 만료 후 액션
- 동시 수정 (다른 탭에서)
- 뒤로가기 후 재제출

---

## 7. 접근성 테스트 (Accessibility)

WCAG 가이드라인 기반 테스트입니다.

### 체크리스트

- [ ] 키보드만으로 모든 기능 접근 가능
- [ ] 포커스 표시 명확
- [ ] 이미지에 alt 텍스트
- [ ] 색상 대비 4.5:1 이상
- [ ] 스크린리더 호환

---

## 8. 성능 테스트 케이스

응답 시간과 부하 테스트입니다.

### 응답 시간 기준

| 작업 | 목표 | 임계 |
|------|------|------|
| 페이지 로딩 | < 2초 | < 5초 |
| API 응답 | < 500ms | < 2초 |
| 검색 결과 | < 1초 | < 3초 |

### 부하 테스트

- 동시 사용자 100명
- 초당 요청 1000회
- 피크 타임 시뮬레이션
