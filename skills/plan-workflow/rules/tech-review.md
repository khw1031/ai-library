# Tech Review - 기술 관점 검토

> 백엔드 + 데이터베이스 + 보안 전문가 관점

---

## Critical 레드플래그

다음 패턴 발견 시 **즉시 보고**:

### Backend
| 패턴 | 문제 | 질문 |
|------|------|------|
| "실시간" | 정의 모호 | 몇 초 이내? |
| 무제한 목록 | 성능 이슈 | 페이지네이션? |
| 동시 수정 | 데이터 충돌 | 락 정책? |
| 상태 전이 | 불완전 | 모든 조건 정의? |

### Database
| 패턴 | 문제 | 질문 |
|------|------|------|
| 무제한 데이터 | 스토리지 | 보관/삭제 정책? |
| N:M 관계 | 조인 성능 | 비정규화 고려? |
| 히스토리 없음 | 추적 불가 | 변경 이력 필요? |

### Security
| 패턴 | 문제 | 질문 |
|------|------|------|
| 권한 미정의 | 접근 제어 누락 | 역할별 권한? |
| 민감 데이터 | 보호 필요 | 암호화 범위? |
| 입력 검증 없음 | 삽입 공격 | 검증 규칙? |

---

## 검토 체크리스트

### API 설계
- [ ] 필요한 엔드포인트가 도출되는가?
- [ ] 요청/응답 형식이 명확한가?
- [ ] 에러 응답이 정의되었는가?
- [ ] 벌크 처리가 필요한 작업은?

### 비즈니스 로직
- [ ] 비즈니스 규칙이 명확한가?
- [ ] 예외 케이스 처리가 정의되었는가?
- [ ] 트랜잭션 범위가 명확한가?
- [ ] 계산 로직(가격, 할인 등)이 구체적인가?

### 데이터 모델
- [ ] 핵심 엔티티가 식별되는가?
- [ ] 관계(1:1, 1:N, N:M)가 명확한가?
- [ ] 인덱스가 필요한 필드는?
- [ ] soft delete vs hard delete?

### 보안
- [ ] 인증 방식이 정의되었는가?
- [ ] 역할별 접근 권한이 명확한가?
- [ ] 민감 데이터가 식별되었는가?
- [ ] 입력 검증 규칙이 있는가?
