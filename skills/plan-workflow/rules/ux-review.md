# UX Review - 사용자 경험 관점 검토

> UI/UX + User Flow 전문가 관점

---

## Critical 레드플래그

다음 패턴 발견 시 **즉시 보고**:

### UI/UX
| 패턴 | 문제 | 질문 |
|------|------|------|
| 막다른 화면 | 다음 행동 불명확 | 다음 단계는? |
| 숨겨진 기능 | 발견 어려움 | 진입점 추가? |
| 모호한 레이블 | 의미 불명확 | 명확한 텍스트? |
| 과도한 단계 | 이탈 위험 | 단축 가능? |

### User Flow
| 패턴 | 문제 | 질문 |
|------|------|------|
| 순환 루프 | 무한 반복 | 탈출 조건? |
| 누락된 상태 | 사용자 혼란 | 로딩/에러/빈 상태? |
| 끊어진 체인 | 연결 불명확 | 이전/다음 화면? |
| 숨겨진 전제 | 암묵적 조건 | 명시적 안내? |

---

## 검토 체크리스트

### 사용성
- [ ] 핵심 기능까지 몇 번의 클릭?
- [ ] 사용자가 현재 위치를 아는가?
- [ ] 실수를 되돌릴 수 있는가?
- [ ] 확인/취소 버튼 위치가 일관적인가?

### 일관성
- [ ] 같은 기능이 같은 위치에 있는가?
- [ ] 용어가 일관되게 사용되는가?
- [ ] 버튼/아이콘 스타일이 일관적인가?

### 피드백
- [ ] 모든 액션에 피드백이 있는가?
- [ ] 로딩 상태가 표시되는가?
- [ ] 에러 메시지가 해결 방법을 안내하는가?
- [ ] 성공 상태가 명확한가?

### 화면 상태
- [ ] 빈 상태(첫 사용자) 화면이 정의되었는가?
- [ ] 에러 상태 화면이 정의되었는가?
- [ ] 로딩 상태가 정의되었는가?
- [ ] 데이터가 많을 때 화면은?

### 사용자 여정
- [ ] 핵심 목표 달성 경로가 완전한가?
- [ ] Happy Path가 명확한가?
- [ ] 예외 시나리오가 정의되었는가?
- [ ] 이탈/재진입 시 동작은?

## Nielsen's 10 Heuristics

| # | 원칙 | 체크 |
|---|------|------|
| 1 | 가시성 | 시스템 상태가 보이는가? |
| 2 | 현실 일치 | 익숙한 용어/개념인가? |
| 3 | 제어와 자유 | 취소/되돌리기가 있는가? |
| 4 | 일관성 | 같은 것이 같아 보이는가? |
| 5 | 오류 예방 | 실수를 방지하는가? |
| 6 | 인식 > 기억 | 보면 알 수 있는가? |
| 7 | 유연성 | 숙련자 단축키가 있는가? |
| 8 | 미니멀 | 불필요한 정보는 없는가? |
| 9 | 오류 복구 | 에러 해결을 안내하는가? |
| 10 | 도움말 | 필요시 도움을 받을 수 있는가? |
