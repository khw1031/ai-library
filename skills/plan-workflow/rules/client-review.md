# Client Review - 클라이언트 관점 검토

> Frontend + App(Mobile) 전문가 관점

---

## Critical 레드플래그

다음 패턴 발견 시 **즉시 보고**:

### Frontend
| 패턴 | 문제 | 질문 |
|------|------|------|
| 무한 스크롤 + 필터 | 복잡한 상태 | 초기화 정책? |
| 실시간 + 오프라인 | 동기화 복잡 | 충돌 해결? |
| 복잡한 폼 | 상태 관리 | 중간 저장? |
| 대량 테이블 | 성능 이슈 | 가상화 필요? |

### App (Mobile)
| 패턴 | 문제 | 질문 |
|------|------|------|
| 디지털 상품 결제 | **정책 위반** | 인앱 결제 필수! |
| 위치 항상 추적 | 배터리/권한 | 필수 여부? |
| 대용량 다운로드 | 데이터 요금 | WiFi 제한? |
| 백그라운드 작업 | iOS 제한 | 대안 필요? |
| 웹뷰 과다 | 심사 위험 | 네이티브화? |

## 검토 체크리스트

### 컴포넌트
- [ ] 재사용 가능한 컴포넌트가 식별되는가?
- [ ] 컴포넌트 계층이 명확한가?
- [ ] 공통/특화 컴포넌트가 구분되는가?

### 상태 관리
- [ ] 전역/로컬 상태 구분이 명확한가?
- [ ] 서버 상태 동기화 방법은?
- [ ] 낙관적 업데이트가 필요한 곳은?

### 데이터 페칭
- [ ] API 호출 시점이 명확한가?
- [ ] 로딩/에러/성공 상태가 정의되었는가?
- [ ] 캐싱이 필요한 데이터는?
- [ ] 실시간 업데이트가 필요한 데이터는?

### 반응형 (Web)
- [ ] 지원 화면 크기가 정의되었는가?
- [ ] 브레이크포인트별 변화가 명시되었는가?
- [ ] 모바일/데스크톱 레이아웃 차이는?

### 플랫폼 (App)
- [ ] iOS/Android 지원 범위?
- [ ] 최소 지원 OS 버전?
- [ ] 필요한 권한 목록?
- [ ] 권한 거부 시 대체 흐름?

### 오프라인 (App)
- [ ] 오프라인 기능 범위?
- [ ] 로컬 저장 데이터?
- [ ] 동기화 충돌 해결 정책?

## 플랫폼별 주의사항

### Web
- SEO 중요 페이지 → SSR/SSG 필요
- 초기 로딩 최적화 → 코드 스플리팅
- 에러 바운더리 위치 정의

### iOS
- **인앱 결제 필수** (디지털 상품)
- 노치/다이나믹 아일랜드 대응
- 백그라운드 제한 엄격
- App Tracking Transparency

### Android
- 다양한 화면 크기 대응
- 뒤로가기 버튼 동작 정의
- 권한 요청 시점 중요
- 폴더블 디바이스 고려
