# 웹 검색 보강 가이드

문서 통합 시 부족한 내용을 웹 검색으로 보강하는 상세 가이드입니다.

## 1. 보강 필요 영역 식별

### 자동 감지 패턴

```
보강 필요 신호:
├── "~에 대해서는 추후 확인 필요"
├── "자세한 내용은 별도 참조"
├── 정의 없이 사용된 전문 용어
├── 구체적 수치/데이터 없는 주장
├── 오래된 날짜의 정보 (1년 이상)
└── "일반적으로", "보통" 등 모호한 표현
```

### 보강 우선순위

| 우선순위 | 유형 | 설명 |
|----------|------|------|
| 높음 | 핵심 개념 정의 | 문서 이해에 필수적인 용어 |
| 높음 | 최신 데이터 | 통계, 버전, 가격 등 |
| 중간 | 배경 설명 | 컨텍스트 이해 보조 |
| 낮음 | 추가 예시 | 이해도 향상용 |

## 2. 웹 검색 수행

### 검색 쿼리 작성

```
효과적인 검색 패턴:
├── "[용어] definition" - 정의 검색
├── "[주제] official documentation" - 공식 문서
├── "[기술] best practices 2024" - 최신 권장사항
├── "[통계] statistics latest" - 최신 통계
└── "[제품] release notes" - 버전 정보
```

### 신뢰도 평가 기준

| 신뢰도 | 출처 유형 | 예시 |
|--------|----------|------|
| 최상 | 공식 문서 | docs.python.org, developer.mozilla.org |
| 상 | 학술/연구 | arxiv.org, scholar.google.com |
| 상 | 신뢰 매체 | techcrunch.com, wired.com |
| 중 | 커뮤니티 | stackoverflow.com, reddit.com |
| 하 | 개인 블로그 | 검증 필요, 보조 자료로만 사용 |

## 3. 보강 내용 통합

### 인라인 보강 형식

```markdown
원본 내용과 자연스럽게 연결:

Docker는 컨테이너 기반 가상화 플랫폼입니다.
**[보강]** 컨테이너는 애플리케이션과 그 종속성을 패키지화하여
어떤 환경에서도 일관되게 실행할 수 있게 해주는 기술입니다 [^1].

[^1]: [Docker Docs - What is a container?](https://docs.docker.com/get-started/overview/)
```

### 별도 박스 형식

```markdown
> 📚 **보강 정보**
>
> Kubernetes의 최신 안정 버전은 v1.29입니다 (2024년 1월 기준).
> 주요 변경사항: 사이드카 컨테이너 지원, 향상된 보안 기능 등.
>
> *출처: [Kubernetes Release Notes](https://kubernetes.io/releases/)*
```

## 4. References 섹션 작성

### 구조

```markdown
## References

### 원본 문서
1. **프로젝트 기획서** - 2024.01.15, 기획팀
2. **기술 스펙 문서** - 2024.01.20, 개발팀

### 웹 검색 출처

#### 개념 정의
[^1]: [Docker Documentation](https://docs.docker.com/) - 컨테이너 개념 정의
[^2]: [Kubernetes Docs](https://kubernetes.io/docs/) - 오케스트레이션 설명

#### 최신 데이터
[^3]: [State of JS 2023](https://stateofjs.com/) - 프레임워크 사용 통계
[^4]: [Stack Overflow Survey 2023](https://survey.stackoverflow.co/) - 개발자 트렌드

#### 보조 자료
[^5]: [Martin Fowler's Blog](https://martinfowler.com/) - 아키텍처 패턴 설명
```

### 출처 정보 필수 요소

```
각 출처에 포함할 정보:
├── 출처 제목/사이트명
├── URL (클릭 가능한 링크)
├── 해당 출처에서 참조한 내용 요약
└── (선택) 접근 날짜, 저자
```

## 5. 보강 품질 체크리스트

```
□ 보강 내용이 원본과 자연스럽게 연결되는가?
□ 모든 보강 내용에 출처가 명시되어 있는가?
□ 출처 URL이 유효하고 접근 가능한가?
□ 신뢰할 수 있는 출처를 사용했는가?
□ 보강 내용이 원본의 맥락과 일치하는가?
□ References 섹션이 체계적으로 정리되어 있는가?
□ 인라인 참조 번호와 References가 매칭되는가?
```

## 6. 보강하지 않는 경우

다음 상황에서는 웹 검색 보강을 수행하지 않습니다:

- 사용자가 `보강 수준: none` 옵션 지정
- 내부 기밀 정보가 포함된 문서
- 이미 충분히 완전한 문서
- 검색으로 찾을 수 없는 내부 용어/프로젝트명
