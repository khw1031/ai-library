# 문서 통합 전략 상세 가이드

## 1. 문서 분석 프로세스

### 1.1 개별 문서 분석
```
각 문서에서 추출할 정보:
├── 주제 (Topic)
├── 핵심 키워드
├── 문서 유형 (보고서, 매뉴얼, 회의록 등)
├── 대상 독자
└── 핵심 주장/결론
```

### 1.2 문서 간 관계 분석
- **보완 관계**: 서로 다른 관점에서 같은 주제 설명
- **확장 관계**: 한 문서가 다른 문서의 세부 내용
- **병렬 관계**: 독립적인 주제들
- **충돌 관계**: 상반된 정보 (명시적 표시 필요)

### 1.3 보강 필요 영역 식별
```
보강 필요 신호 감지:
├── 정의 없는 전문 용어
├── 불완전한 설명 ("추후 확인", "별도 참조")
├── 오래된 정보 (날짜, 버전, 통계)
├── 모호한 표현 ("일반적으로", "보통")
└── 구체적 근거 없는 주장
```

## 2. 웹 검색 보강 프로세스

### 2.1 보강 대상 결정
```
우선순위 판단:
├── [필수] 핵심 개념 정의 누락
├── [필수] 오래된 핵심 데이터
├── [권장] 배경 지식 보충
└── [선택] 추가 예시/참고자료
```

### 2.2 검색 및 수집
1. 보강 필요 키워드로 웹 검색 수행
2. 신뢰할 수 있는 출처 우선 선택
3. 관련 내용과 URL 수집

### 2.3 보강 내용 통합
```markdown
원본 텍스트...

**[보강]** 웹 검색으로 확인된 추가 정보 [^출처번호].

[^출처번호]: [출처명](URL) - 참조 내용 요약
```

## 3. 구조화 패턴

### 2.1 주제 중심 구조
```markdown
# 통합 문서

## Executive Summary
## 주제 A
### 개요 (문서1 기반)
### 상세 (문서2 기반)
## 주제 B
...
```

### 2.2 시간순 구조
```markdown
# 통합 문서

## Executive Summary
## 배경/이전 상황
## 현재 상태
## 향후 계획
```

### 2.3 계층적 구조
```markdown
# 통합 문서

## Executive Summary
## 상위 개념
### 하위 개념 A
### 하위 개념 B
## 세부 사항
```

## 3. Executive Summary 작성법

### 필수 포함 요소
1. **목적**: 통합 문서가 다루는 범위
2. **핵심 내용**: 3-5개의 주요 포인트
3. **결론/권고**: 액션 아이템 또는 결론
4. **키워드**: 검색/참조용 핵심 용어

### 작성 템플릿
```markdown
## Executive Summary

**목적**: [통합 문서의 목적 1문장]

**핵심 내용**:
- [핵심 포인트 1]
- [핵심 포인트 2]
- [핵심 포인트 3]

**주요 키워드**: [키워드1], [키워드2], [키워드3]

**원본 문서**: [문서 수]개 문서 통합
```

## 4. 중복 처리 전략

### 4.1 완전 중복
- 하나만 유지, 출처 병기

### 4.2 부분 중복
- 더 상세한 버전 유지
- 추가 정보는 보충

### 4.3 유사하지만 다른 내용
- 둘 다 유지
- 차이점 명시

### 4.4 충돌하는 내용
```markdown
> **참고**: 다음 내용은 문서 간 차이가 있습니다.
> - 문서 A: [내용]
> - 문서 B: [내용]
```

## 5. 출처 표시 방법

### 인라인 표시
```markdown
프로젝트 일정은 3개월입니다 [문서1].
```

### 섹션 말미 표시
```markdown
### 기술 스펙
...내용...

*출처: 기술 스펙 문서, 2024.01*
```

### 참고 섹션
```markdown
## 참고

### 원본 문서 목록
1. [문서명1] - 작성일, 작성자
2. [문서명2] - 작성일, 작성자

### 통합 정보
- 통합일: YYYY-MM-DD
- 통합자: [이름/시스템]
```

## 6. 품질 체크리스트

```
통합 품질:
□ Executive Summary가 전체 내용을 대표하는가?
□ 논리적 흐름이 자연스러운가?
□ 중복이 적절히 처리되었는가?
□ 충돌하는 정보가 명확히 표시되었는가?
□ 목차가 내용을 잘 반영하는가?
□ 일관된 용어와 형식이 사용되었는가?

웹 검색 보강 품질:
□ 보강이 필요한 영역이 모두 식별되었는가?
□ 신뢰할 수 있는 출처를 사용했는가?
□ 모든 보강 내용에 출처 링크가 있는가?
□ 보강 내용이 원본 맥락과 일치하는가?
□ References 섹션이 체계적으로 정리되었는가?
□ 인라인 참조 번호와 References가 매칭되는가?
```
