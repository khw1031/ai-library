# First Principles Thinking Framework

주어진 문제/질문/주제에 대해 가정을 해체하고, 근본 원리만 추출하여 새로운 해결책을 재구성하는 분석 도구입니다.

## 시뮬레이션 목표

4명의 전문가 관점에서 문제를 해체하고 근본 진실로부터 해결책을 재구축합니다.

| 전문가 | 역할 | 핵심 질문 |
|--------|------|-----------|
| Physicist | 근본 원리 분석 | 물리 법칙과 논리적 필연성만 남기면 무엇이 남는가? |
| Socratic Philosopher | 가정 해체 | 이것이 정말 "당연한" 것인가? 왜? |
| Systems Engineer | 재구성 | 근본 요소들로 어떤 새로운 시스템을 구성할 수 있는가? |
| Domain Specialist | 맥락적 적용 | 이 분야의 실제 제약과 기회는 무엇인가? |

각 전문가는 내부적으로 다음을 검토합니다:

- 이 문제에서 "당연하다"고 여겨지는 가정들은 무엇인가?
- 물리 법칙, 논리적 필연성, 또는 증명된 사실만 남기면 무엇이 남는가?
- 이 근본 요소들로부터 어떤 새로운 해결책을 구성할 수 있는가?

## 분석 프로세스

### Phase 1: Assumption Excavation (가정 발굴)

입력된 [문제/주제]에 대해 다음을 수행합니다:

1. 현재 "업계 표준", "상식", "원래 그런 것"으로 받아들여지는 가정 5-7개 나열
2. 각 가정의 출처를 분류

| 분류 | 의미 | 처리 |
|------|------|------|
| 관습 | 관행적으로 그래왔을 뿐, 필연적 근거 없음 | 해체 대상 |
| 기술적 제약 | 현재 기술 수준의 한계 (변경 가능) | 조건부 해체 |
| 물리 법칙 | 자연 법칙으로 변경 불가 | 유지 |
| 검증된 사실 | 실험/데이터로 증명된 사실 | 유지 |

3. "관습"으로 분류된 항목에 경고 표시 -- 이것이 Phase 2의 해체 대상

### Phase 2: Fundamental Decomposition (근본 분해)

경고 표시된 가정들에 대해 "5 Whys" 기법을 적용합니다:

```
가정: "X는 반드시 Y여야 한다"
  왜? -> 이유 1
  왜? -> 이유 2
  왜? -> 이유 3
  왜? -> 이유 4
  왜? -> [더 이상 분해 불가능한 근본 요소 도출]
```

각 근본 요소(First Principle)에 대해:

- 근본 요소 명확히 서술
- 해당 요소가 참인 증거/근거 제시
- 이 요소가 기존 가정과 어떻게 다른지 대비

### Phase 3: Ground-Up Reconstruction (재구성)

추출된 근본 요소들만으로 새로운 해결책을 구성합니다:

1. 기존 접근법과 다른 해결 경로 최소 3개 도출
2. 각 경로의 장단점 분석
3. "왜 기존에는 이렇게 하지 않았는가?"에 대한 가설 제시

## 출력 형식

분석 결과는 반드시 한국어로 다음 형식에 따라 출력합니다.

```markdown
# First Principles 분석: [문제/주제]

## 가정 해부

| 기존 가정 | 분류 | 근본 진실 |
|-----------|------|-----------|
| [가정 1] | 관습 | [해체 후 도출된 근본 진실] |
| [가정 2] | 물리 법칙 | [변경 불가 - 그대로 유지] |
| [가정 3] | 관습 | [해체 후 도출된 근본 진실] |
| ... | ... | ... |

## First Principles 목록

1. [근본 원리 1] - 근거: ...
2. [근본 원리 2] - 근거: ...
3. [근본 원리 3] - 근거: ...

## 재구성된 해결책

### Option A: [기존 관습을 깬 접근법 이름]

- **근거**: [어떤 First Principle에서 도출되었는지]
- **장점**: ...
- **단점**: ...
- **실행 난이도**: 상/중/하

### Option B: [접근법 이름]

- **근거**: ...
- **장점**: ...
- **단점**: ...
- **실행 난이도**: 상/중/하

### Option C: [접근법 이름]

- **근거**: ...
- **장점**: ...
- **단점**: ...
- **실행 난이도**: 상/중/하

## 실행 권고

[가장 현실적이면서 혁신적인 경로 제안]

**첫 번째 실행 단계**: [구체적이고 즉시 실행 가능한 행동]
```

/first-principles 왜 회의는 항상 비효율적인가?
